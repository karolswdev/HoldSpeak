Dictation Runtime Setup

The dictation pipeline uses a local LLM runtime for block classification and Project KB enrichment. Pick one backend and place the matching model at the configured path.

Apple Silicon

Use MLX on Apple Silicon when available.

uv pip install -e '.[dictation-mlx]'

huggingface-cli download mlx-community/Qwen3-8B-MLX-4bit --local-dir ~/Models/mlx/Qwen3-8B-MLX-4bit

Default model path: ~/Models/mlx/Qwen3-8B-MLX-4bit

Cross-Platform GGUF

Use llama_cpp on Linux x86_64, or as a macOS fallback. On macOS arm64, build with Metal flags. 

CMAKE_ARGS="-DGGML_METAL=on" uv pip install -e '.[dictation-llama]'

mkdir -p ~/Models/gguf
huggingface-cli download bartowski/Qwen2.5-3B-Instruct-GGUF \
  Qwen2.5-3B-Instruct-Q4_K_M.gguf --local-dir ~/Models/gguf --local-dir-use-symlinks False

Default model path: ~/Models/gguf/Qwen2.5-3B-Instruct-Q4_K_M.gguf

Enable Runtime

Enable the dictation pipeline from the Runtime tab, or set dictation.pipeline.enabled in ~/.config/holdspeak/config.json.

Readiness and doctor only show commands; they do not install packages or download models.
