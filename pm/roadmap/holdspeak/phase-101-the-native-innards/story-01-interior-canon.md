## Problem

Interiors compose as generic kit (AGENT_BRIEF.md §5): label+input
stacks, uniform rows, one text size, verbs in action rows. The owner:
"a bunch of HTML slapped inside a nicer looking container." And the
standing order (§6): "just not be shy — really push this into OS
territory."

## Scope

- In:
  - the INTERIOR TYPE SCALE ratified as tokens (display / primary /
    secondary / caption inside window bodies) + the composition rules
    (data as material, verbs on the material, when a row list is
    legal) written into DESIGN_SYSTEM.md;
  - kit additions that carry them (in-place edit affordance, stream/
    library/switchboard composition primitives) — designed, not yet
    wired product-wide;
  - the OS-territory inventory (§6) ratified alongside: the system
    shade, the global keyboard grammar (⌘1–4 / ⌘W / ⌘M / ⌘/),
    universal context menus, drag-and-drop as a system verb, system
    moments, the Files-grade list;
  - high-fidelity mockups at 1440 and 393, real content, looked at:
    the three worst innards (Speak's Journal as a dated stream,
    Blocks as a library, Runs on as a switchboard) PLUS the system
    shade and one drag-through-the-glass moment.
- Out: shipping interiors (the gate decides).
