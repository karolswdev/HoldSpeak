# HS-93-09 — The owner can live here

- **Project:** holdspeak
- **Phase:** 93
- **Status:** backlog
- **Depends on:** HS-93-01, HS-93-02, HS-93-03, HS-93-04, HS-93-05, HS-93-06, HS-93-07, HS-93-08
- **Unblocks:** none
- **Owner:** unassigned

## Problem

A scripted closeout can prove contracts while missing whether HoldSpeak is
pleasant and dependable as a daily tool. Closure requires sustained use on the
exact production roots, accumulated state, real interruptions, and honest
triage—not a fresh fixture that lasts one afternoon.

## Scope

- **In:** Five working days of owner dogfood across production Web, physical
  iPhone, and physical iPad; daily dictation, meetings, organization, grounded
  ask, Persona/Workflow, destination choice, Integration, Coder, approval/grant,
  recovery, and next-day return; raw measures; defect sweep; bounded automated
  close lanes; docs; Phase-92 criterion reconciliation; evidence and phase close.
- **Out:** Waiving a blocker, crediting simulator/responsive Web as native,
  unrelated features, or creating evidence before execution.
- **Paths:** UAT campaigns/conductor/closeout, dogfood protocols, Python/Web/
  Swift tests, production builds, docs, all Phase-93 evidence, and linked
  Phase-92 criteria.

## Acceptance criteria

- [ ] The owner uses HoldSpeak on five working days with exact commit, build,
      browser, device, OS, model, endpoint, audio route, network, and pairing
      provenance; each day begins from accumulated prior state.
- [ ] The ten primary journeys complete on production Web and flagship Swift,
      with entry, focused work, retained result/failure, receipt, and Desk return
      directly observed.
- [ ] Raw before/after measures cover navigation/control count, first-value
      steps/decisions/time, ambiguous terms/verbs, capture memory/loss, recovery,
      duplicate/silent outcomes, Desk return, and sustained finding rate.
- [ ] The owner copy read-through finds zero unresolved promotional, narrative,
      theatrical, patronizing, anthropomorphic, ambiguous, or redundant
      operational strings across the ten React and flagship Swift journeys.
- [ ] No lost work, false completion, silent fallback/retarget, paired-as-local
      claim, unexplained external effect, drag-only primary action, or unresolved
      blocking finding remains.
- [ ] Secure/Normal/YOLO are each used for real daily work on React and flagship
      Swift; raw prompt counts prove YOLO generated zero HoldSpeak approval
      prompts for eligible configured operations while every invariant failure
      remained blocked and receipted.
- [ ] Every non-pass observation is triaged; fixes are re-run on the affected
      production target, and unrelated follow-ups enter the backlog without
      weakening the phase gate.
- [ ] Canonical bounded Python lanes, `npm run check`, UAT Web tests and site
      walk, Swift package and production app builds, secret/term/architecture
      guards, packaging, and Delivery Workbench checks are captured green.
- [ ] Phase-92 criteria satisfied by this work are individually linked and
      re-checked; Phase 92 receives an honest disposition rather than implied
      success.
- [ ] Phase-93 story evidence, final summary, roadmap status, and owner verdict
      are created only after every required run is complete.

## Test plan

- **Unit:** repository-canonical bounded Python, Web, UAT-site, and Swift suites;
  closeout-policy tests.
- **Integration:** full relevant integration/E2E lanes excluding opt-in metal;
  metal/device lanes separately; Web UI audit; UAT campaign and fail-closed
  closeout on one clean commit.
- **Manual / device:** Five-day owner dogfood with production Web, physical
  iPhone, and physical iPad; real microphones, meetings, destinations,
  Integration, Coder, accessibility, faults, and next-day continuity.

## Notes / open questions

Five working days is a minimum sustained-use window, not permission to ignore a
blocking defect discovered on day five. The story remains open until the rerun.

Bundling note: this initial Phase-93 scaffold is intentionally committed with
the HS-93-01 through HS-93-05 in-progress implementation slices because the
owner directed that the complete shared working tree ship together. No story is
marked done; each closure gate remains independent.
