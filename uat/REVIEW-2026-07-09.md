# UAT protocol and harness review — 2026-07-09

> **Protocol-v2 addendum:** This review is retained as the evidence that exposed
> the original corpus state; its scenario counts, old target names, and blockers
> are historical facts at the time of review. The follow-up owner audit found a
> deeper issue: web/iPad/iPhone mixed implementation identity with form factor.
> Current execution uses `web_react`, `ios_flagship_swift`,
> `ios_companion_swift`, and `ios_classic_swift` plus explicit compatible form
> factors. Responsive React never counts as Swift, native verdicts require a
> matching pairing-verified device attestation, and parity joins independently
> executed target-specific legs.

## Readiness verdict

The rig is useful for structured dogfooding, but it was not trustworthy as a
release-acceptance instrument at the start of this review. Schema tests were
green while several control/treatment transitions never executed, native runs
were loopback-only, authored citations were displayed as sitting coverage, and
old debriefs could change when scenario YAML changed.

This pass closes those harness-integrity defects and adds a current-flagship
device smoke pack. It does **not** declare the whole product accepted. The
remaining blockers are product-target reachability, on-glass inventory
resolution, step mapping across the legacy corpus, per-device verdict identity,
and fresh-state isolation for mutating parity checks.

## Audited surface

- Feature ledger: 254 live capabilities; 115 `must`, 94 `should`, 44 `spot`, 1
  `skip`. All 115 must keys are cited somewhere, but citation is not a pass.
- Protocol after this pass: 10 packs, 109 scenarios, 345 steps, 611 possible
  per-surface verdicts; 33 hand-staged scenarios.
- Execution targets: 94 web scenarios, 9 current-flagship iOS, 4 classic iOS,
  and 2 companion-shell iOS. The target split is now explicit.
- Native inventory audit: current TestFlight/production root is the DioStage
  `HoldSpeakMeetingCapture` target. Meeting aftercare/archive/import/learning
  claims live in a separately built companion shell; dictation teleprompter,
  CommandsBoard, floating recorder/canvas, and Workbench claims are compiled
  but reachable only through a classic/demo root.
- Web inventory audit: the first-run redirect blocks ordinary Desk arrival;
  several scenarios described state controls the shipped UI cannot produce;
  `/cadence` is an actual uncatalogued page; 86 of 220 `web: yes` ledger keys
  remain uncited (mostly `should`/`spot`).

## P0 findings and disposition

1. **Device sittings created loopback-only product runs.** Fixed: the picker has
   Device mode, native packs require it, the pairing URL is visible, the product
   LAN token survives deck restarts, and the CLI prints a usable LAN address.
2. **`after:` transitions were effectively skipped by the guided UI.** Fixed:
   transitions execute server-side after the final applicable verdict, persist
   exactly once, block the next beat on failure, and expose retry/status.
3. **Multi-recipe scenarios silently ran only the last recipe's deck.** Fixed:
   cross-deck initial recipes fail validation; affected protocols now use an
   ordered step transition or a single composite recipe.
4. **Old sittings reloaded mutable current YAML/ledger.** Fixed: each sitting
   snapshots normalized scenarios, ledger, referenced recipe/deck/seed hashes,
   and git commit; resume and debrief read the snapshot.
5. **Debrief coverage meant authored citations, not executed work.** Fixed:
   executed coverage is separate and excludes skipped/unwalked checks. New and
   corrected high-risk scenarios use step-level `verifies` attribution.
6. **Findings were ordinal and stale triage rows survived corrected verdicts.**
   Fixed: one deterministic finding per `(run, scenario, step)`, grouped across
   surfaces; obsolete findings are removed before backlog generation.
7. **Useful application logs were omitted.** Fixed: debrief reads HoldSpeak's
   isolated application log plus stdout/stderr and normalizes UTC verdict time
   to host-local wall time for correlation.
8. **Manual staging/probe outcomes were browser memory only.** Fixed at the
   scenario level: staging, probe results, manual confirmation, and transitions
   persist and are included in `debrief.json`. Per-checklist-item evidence is a
   remaining enhancement.
9. **Observations on otherwise accepted behavior had no lifecycle.** Fixed:
   `observe` is a first-class noted verdict/finding; triage now requires a
   disposition.
10. **Screenshot replacement could retain a stale path.** Fixed; uploads use
    content-addressed names and verdicts accept only artifacts under that run.

## Remaining acceptance blockers

1. **Flagship reachability:** several ledger `ipad: yes` claims remain valid only
   for companion/classic builds, not the installed flagship. Product/roadmap
   must either merge them into the production root or explicitly ship/test the
   separate target.
2. **Device provenance:** the conductor polls verdicts across browsers but does
   not bind an iPhone/iPad verdict to a registered device/build session.
3. **Checklist evidence:** manual preflight confirmation is durable, but its
   individual facts (build, OS, model hash, permissions, audio route) are still
   recorded in the first note rather than typed fields.
4. **Legacy traceability:** only the newly hardened high-risk slice uses
   step-level `verifies`; legacy scenarios still receive scenario-level credit.
5. **Surface isolation:** one run is shared across surfaces. Mutating CRUD
   protocols need per-surface runs or an explicit shared-state/sync contract.
6. **Semantic backlog:** Web Desk audit still identifies impossible/overclaimed
   edges in Qlippy timing, direct-mic coder behavior, per-card egress, and broad
   “transcribe every input” language that need on-product resolution.
7. **Inventory gaps:** `/cadence` and other uncited Web Desk pages need a
   dedicated cockpit pack; the native flagship needs explicit capture, settings,
   profiles, and compact-steering inventory walks beyond the new smoke spine.
